# TRIZ 问题解决路径选择功能 — 设计规格

**日期**: 2026-07-17
**状态**: 待用户审阅
**版本**: v1.0

---

## 1. 概述

为 TRIZ 技能 (TrizInv) 新增问题解决路径选择功能，位于现有 Problem Routing 之前，作为用户使用技能的统一入口。

### 1.1 核心概念

两种问题解决路径，分别由两个不同人格的 AI 执行：

| 路径 | 人格 | 交互模式 | 适用于 |
|------|------|---------|--------|
| **放射型** (Radial) | TRIZ 分析师 | 一次输入，全自动执行，静默输出报告 | 问题边界清晰、有经验的用户 |
| **收敛型** (Convergent) | TRIZ 教练 | 多轮对话，共同定义问题，确认路线图后自动执行 | 问题模糊、新领域、需要对齐认知 |

### 1.2 关键设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 与现有 Problem Routing 的关系 | 前置元层 | 先决定"怎么解"，再进入"用什么工具" |
| 路径选择方式 | 推荐+确认（C） | 系统推荐+简短理由，用户确认或切换 |
| 收敛型终点 | 收敛到问题定义（A） | 产出问题定义书+路线图，确认后自动转入放射型 |
| Triz-AI 专利挖掘 | 必选项 | 放射型、收敛型都必经 Triz-AI 节点 |
| 设计方向 | 双模人格（方案3） | 两种模式有独立人格、交付物、交互协议 |

## 2. 整体架构

```
用户带着问题进来
       │
       ▼
┌─────────────────────────────┐
│     路径推荐网关 (Gateway)    │  ← 新增：自动分析+推荐+用户确认
│  "你的问题边界较清晰 → 放射"  │
└──────────┬──────────────────┘
           │
     ┌─────┴─────┐
     ▼           ▼
┌─────────┐ ┌─────────┐
│ 收敛型    │ │ 放射型    │
│ TRIZ教练 │ │ TRIZ分析师│
└────┬────┘ └────┬────┘
     │           │
     ▼           │
┌─────────┐      │
│问题定义书 │      │
│+ 执行路线图│     │
└────┬────┘      │
     │ 用户确认   │
     ▼           ▼
┌─────────────────────────────┐
│      ★ Triz-AI 专利挖掘      │  ← 必选项：两路都经此节点
│   专利检索 → 竞争分析 → 技术趋势 │
└──────────┬──────────────────┘
           ▼
┌─────────────────────────────┐
│      放射型执行引擎           │
│  选工具 → 分析 → 整合专利 → 报告 │
└──────────┬──────────────────┘
           ▼
     ┌──────────┐
     │ 输出报告   │
     │ + PPT(可选)│
     └──────────┘
```

### 2.1 设计原则

- **网关是统一入口**：不改变用户的调用方式，用户只需描述问题
- **收敛型教练**产出两个交付物：问题定义书（可独立使用）+ 执行路线图（桥接到放射型）
- **放射型分析师**是收敛型的下游，也是独立入口
- **两种人格**各有独立的 system prompt 指令块，互不污染
- **Triz-AI 是必经节点**：无论哪条路径，都必须经过专利挖掘

## 3. 路径推荐网关

### 3.1 判断逻辑

分析用户输入，给出路径推荐：

- **倾向放射型**：问题描述包含具体技术参数、明确的矛盾描述、清晰的改进目标
- **倾向收敛型**：问题仅有时方向性描述、核心矛盾不明确、缺少关键约束信息

### 核心问题

系统评估这个问题是否满足："问题边界清晰 + 核心矛盾明确 + 约束条件已知"

- **是** → 推荐放射型，"你的问题边界比较清晰，建议走自动分析路径。要开始吗？还是想先讨论一下？"
- **否** → 推荐收敛型，"你的问题还需要进一步明确，建议我们先讨论一下。要开始对话吗？还是你直接试试自动分析？"

### 3.2 交互模式

- 推荐结果以简短理由呈现
- 用户确认推荐路径，或手动切换
- 确认后进入对应人格

## 4. 收敛型 — TRIZ 教练

### 4.1 人格特征

- **Socratic 式对话者**：通过提问帮助用户看清问题本质
- **TRIZ 教育者**：在对话中自然引入 TRIZ 概念（矛盾、IFR、资源分析）
- **耐心、引导性**：不急于跳转到工具推荐，确保问题定义扎实

### 4.2 对话结构（4 阶段，每阶段以目标达成为出口）

```
阶段1: 问题初探
目标: 搞清楚"谁、什么事、为什么现在"
出口条件: 能用一句话说清问题场景
├── 你的领域/产品是什么？
├── 你遇到了什么具体问题？
└── 为什么现在要解决它？
    ↳ 用户能清晰描述 → 进入阶段2

阶段2: TRIZ视角深挖
目标: 识别技术矛盾，建立IFR
出口条件: 至少识别出一对明确的矛盾参数，IFR清晰定义
├── 有没有技术矛盾？什么变好了、什么变坏了？
├── 理想最终结果(IFR)是什么？
├── 已尝试过哪些方案？为什么不够好？
└── 约束条件有哪些？
    ↳ 矛盾+IFR都明确 → 进入阶段3

阶段3: 边界收敛
目标: 锁定问题边界，消除模糊地带
出口条件: 用户确认"这就是我要解决的问题"
├── 问题的核心是什么？（提炼一句话）
├── 这个边界对吗？要收窄还是扩展？
└── 还有什么不确定的？
    ↳ 用户确认边界 → 进入阶段4

阶段4: 输出路线图
目标: 展示完整执行计划，获得用户确认
出口条件: 用户确认路线图，选择"执行"
├── 展示问题定义书
├── 展示执行路线图（含Triz-AI检索方向）
└── "确认执行？还是需要调整？"
    ↳ 用户确认 → 进入Triz-AI → 放射型执行引擎
```

### 4.3 交付物 A：问题定义书

```markdown
# 问题定义书

## 核心矛盾
<一句话：什么 vs 什么>

## 理想最终结果 (IFR)
<理想状态描述>

## 问题边界
- 范围: <明确什么在范围内>
- 排除: <明确什么不在范围内>

## 约束条件
- <约束1>
- <约束2>

## 已尝试方案
| 方案 | 为什么不够好 |
|------|-------------|
| ...  | ...         |

## 关键假设
- <需要验证的假设>
```

### 4.4 交付物 B：执行路线图

```markdown
# 执行路线图

## 推荐工具序列
| 顺序 | 工具 | 用途 | 输入 → 输出 |
|------|------|------|-------------|
| 1    | ...  | ...  | ...         |

## Triz-AI 专利检索方向
- 关键词: <提取的关键词组合>
- 技术领域: <IPC/CPC分类建议>
- 竞争关注: <需要重点分析的方向>

## 预计分析深度
- 核心分析: <工具数量> 个工具
- 辅助分析: <工具数量> 个工具
```

### 4.5 异常处理

- 用户在阶段1-3任何时候可以要求跳转到放射型（带着当前已明确的信息直接执行）
- 如果用户无法给出矛盾或IFR（阶段2卡住），教练切换到更引导性的提问模式
- 用户可随时暂停对话，问题定义书自动保存，后续可恢复

## 5. 放射型 — TRIZ 分析师

### 5.1 入口场景

- **直接进入**：网关推荐放射型，用户确认
- **转接进入**：收敛型确认路线图后转入

### 5.2 执行流程

```
用户输入（问题已明确）
       │
       ▼
┌─────────────────────────────┐
│   问题解析 & 工具路由          │  ← 复用现有 11 题 Problem Routing
│   自动匹配工具序列             │
└──────────┬──────────────────┘
           ▼
┌─────────────────────────────┐
│   ★ Triz-AI 专利挖掘          │  ← 必选项
│   自动提取关键词 + 检索        │
└──────────┬──────────────────┘
           ▼
┌─────────────────────────────┐
│   工具执行引擎                 │  ← 现有 6-step 流程
│   按序列执行TRIZ工具分析        │
│   融合专利洞察                 │
└──────────┬──────────────────┘
           ▼
┌─────────────────────────────┐
│   报告生成                    │  ← 完整分析报告
│   + PPT (可选，NotebookLM)    │
└─────────────────────────────┘
```

### 5.3 人格特征

- **静默执行**：中间过程不打断用户，只汇报关键里程碑
- **全自动**：不向用户追问，缺失信息用合理假设填补并标注
- **一次性输出**：最终交付完整报告，用户审阅后可选择深度展开某个部分

### 5.4 里程碑通知

执行过程中，向用户发送简短进度通知：

- "✅ 工具路由完成 — 选用: 矛盾矩阵、功能分析、76标准"
- "🔍 Triz-AI 专利检索完成 — 找到 23 件相关专利"
- "📊 分析进行中 — 已完成 3/5 个工具"
- "📝 报告生成中..."

## 6. 两种模式对比

| | 收敛型 TRIZ教练 | 放射型 TRIZ分析师 |
|---|---|---|
| **互动模式** | 多轮对话 | 一次输入，静默执行 |
| **问题定义** | 由对话共同构建 | 用户自带，自动验证 |
| **工具选择** | 展示路线图，用户确认 | 自动路由，无需确认 |
| **中间产出** | 问题定义书 + 路线图 | 里程碑通知 |
| **Triz-AI** | 检索方向由路线图指定 | 检索方向自动提取 |
| **最终产出** | 转入放射型 → 报告 | 完整报告 |
| **适用场景** | 问题模糊、新领域、需要对齐认知 | 问题清晰、有经验用户、批量分析 |
| **用户参与度** | 高（对话共建） | 低（只需初始输入+审阅结果） |

- [ ] 收敛型转放射型的切换是否无缝？
