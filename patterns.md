## Part 1: The 40 Inventive Principles

Each principle includes its key "flavours" (sub-suggestions). When you identify a Technical or Physical Contradiction, use the Contradiction Matrix (see [cheatsheet.md](cheatsheet.md)) to find which principles to apply first.

### 1. Segmentation — Divide into independent parts; make easy to disassemble; increase fragmentation
### 2. Taking Out — Extract the disturbing part/property; extract only the necessary part
### 3. Local Quality — Non-uniform structure; each part works in its best-suited conditions; each part fulfills a different function
### 4. Asymmetry — Change symmetrical to asymmetrical; if already asymmetrical, increase the asymmetry
### 5. Merging — Merge identical/similar objects or operations; merge parallel operations in time
### 6. Multi-Function — Make one object perform multiple functions; eliminate other objects
### 7. Nested Doll — Place one object inside another; make one part pass through a cavity in another
### 8. Anti-Weight — Compensate with lift; use aero/hydrodynamic forces
### 9. Preliminary Anti-Action — Pre-load an anti-action; create pre-stresses in the opposite direction
### 10. Preliminary Action — Perform before needed; pre-arrange for convenient deployment
### 11. Beforehand Cushioning — Prepare emergency means for low reliability
### 12. Equipotentiality — Limit position changes in a potential field; avoid raising/lowering
### 13. The Other Way Around — Invert the action; make movable parts fixed and fixed parts movable; turn upside down
### 14. Spheroidality — Curved/roller parts instead of linear; use centrifugal force
### 15. Dynamism — Adjustable for different conditions; divide into parts capable of relative movement
### 16. Partial or Excessive Action — Do slightly less or slightly more, then adjust
### 17. Another Dimension — 1D→2D→3D; tilt/re-orient; use the other side
### 18. Mechanical Vibration — Oscillate; use resonant frequency; piezoelectric vibrators
### 19. Periodic Action — Pulse instead of continuous; vary frequency; use pauses for other actions
### 20. Continuity of Useful Action — Eliminate idle time; all parts working all the time
### 21. Skipping — Conduct harmful processes at very high speed
### 22. Blessing in Disguise — Convert harm into benefit; combine two harms to cancel; amplify harm until harmless
### 23. Feedback — Introduce feedback; if exists, reverse it or change magnitude
### 24. Intermediary — Use an intermediary object; temporarily merge with easily removed object
### 25. Self-Service — Object serves itself; use waste resources/energy
### 26. Copying — Inexpensive copies; optical copies; IR/UV copies
### 27. Cheap Short-Living Objects — Replace expensive durable with cheap disposable
### 28. Mechanics Substitution — Sensory systems replace mechanical; use E/M fields; static→moving fields
### 29. Pneumatics & Hydraulics — Gas/liquid instead of solid parts
### 30. Flexible Shells & Thin Films — Replace 3D with flexible shells; isolate with films
### 31. Porous Materials — Make porous; fill pores with useful substance
### 32. Colour Changes — Colour change; transparency change; coloured additives for visibility
### 33. Homogeneity — Interacting objects from same/similar material
### 34. Discarding & Recovering — Used-up parts go away; restore parts during operation
### 35. Parameter Changes — Physical state change; concentration/consistency; flexibility/temperature
### 36. Phase Transitions — Phenomena during phase changes (latent heat, expansion)
### 37. Thermal Expansion — Use expansion/contraction; multiple materials with different CTEs
### 38. Strong Oxidants — Enriched/pure oxygen; ionized oxygen; ionizing radiation
### 39. Inert Atmosphere — Inert environment; neutral additives
### 40. Composite Materials — Homogeneous→composite; combine for properties no single material has

## Part 2: The 8 Trends of Technical Evolution

### Trend 1: S-Curve Evolution
Infancy (invest, protect IP) → Growth (optimize, scale) → Maturity (cost-reduce, trim, add features) → Decline (transition to next-gen)

### Trend 2: Increasing Ideality
Systems deliver more benefit with less cost and harm over time.

### Trend 3: Non-Uniform Sub-system Development
Sub-systems evolve at different rates; the most constrained limits overall performance — innovate there.

### Trend 4: Increasing Dynamism & Controllability
Rigid → hinged → multi-jointed → fully flexible → field-based.

### Trend 5: Complexity then Simplification
Systems gain complexity, then simplify through trimming and integration.

### Trend 6: Matching & Mismatching
Evolve toward optimal matching for critical interfaces; deliberate mismatching where beneficial.

### Trend 7: Transition to Micro-level
Macro → micro → nano → molecular/atomic scale.

### Trend 8: Increasing Automation
Human-operated → semi-automatic → fully automatic → self-learning/adaptive.

## Part 4: ARIZ — Algorithm of Inventive Problem Solving (Ch17)

Gadd's simplified 5-step ARIZ for practitioners:

**Step 1 — Problem Restatement:** Restate the problem in user's words → identify what needs to change → define the Ideal Final Result (IFR) → identify constraints → write the Mini-Problem ("The system for X consists of A,B,C. With minimum changes, the system must provide Y. Find the X-Element that delivers Y without complicating the system.")

**Step 2 — Contradiction Identification:** Extract the Technical Contradiction (if A improves, B worsens) → deepen to Physical Contradiction (X must be BIG and SMALL simultaneously) → use the Contradiction Matrix for initial principle suggestions.

**Step 3 — Resources & Zone of Conflict:** Map Zone of Conflict (ZoF) — the exact spatial location of the contradiction → define Operative Time (OT) — when the conflict occurs → inventory Substance-Field Resources (SFR) in the ZoF at the OT (substances, fields, space, time, information, functions already present).

**Step 4 — Solution Search:** Apply 40 Principles → Separation Principles → Effects Database → 76 Standards → Trends of Evolution → Smart Little People → Knowledge/analogues from other industries. Use the X-Element concept: "There exists an X that solves this without complicating the system. What is X?"

**Step 5 — Solution Refinement:** Evaluate concepts → develop the strongest → resolve secondary problems → test and implement. If no solution found, return to Step 1 and relax constraints.

**When ARIZ:** Problems where no contradiction is obvious; Level 3-4 problems; problems that have resisted simpler TRIZ tools. **When NOT:** Simple trade-offs (use Matrix directly); Level 1-2 problems.

## Part 5: AFI — Anticipatory Failure Identification (Ch18)

4-step process to find and fix failures BEFORE they happen:

**Step 1 — Invert:** "Everything that CAN go wrong, WILL go wrong." List every possible failure mode without filtering.
**Step 2 — Exaggerate:** Amplify each failure. "The bolt loosens" → "The bolt falls out instantly on first use." This forces recognition of failure mechanisms.
**Step 3 — Generate with TRIZ:** For each exaggerated failure, use TRIZ tools (40 Principles, Effects, Standards, Trends) to generate solutions that prevent or mitigate it.
**Step 4 — Re-invert & Fix:** Return to realistic perspective. Prioritize failures by severity × likelihood. Fix the high-priority items using generated solutions.

**When AFI:** Before detailed design; before production ramp-up; when safety/reliability critical. **When NOT:** After failures have already occurred (use RCA); for non-engineering problems.

## Part 6: 12 Double Business Principles (Ch19)

Each business principle is paired with its anti-principle — both are valid in different contexts:

1. **Segmentation ↔ Merging** — Divide vs consolidate markets, teams, products
2. **Taking Out ↔ Adding In** — Outsource vs insource; minimal vs full-featured
3. **Local Quality ↔ Standardization** — Customize vs standardize; adapt vs unify
4. **Asymmetry ↔ Symmetry** — Specialize vs generalize; differentiate vs commoditize
5. **Merging ↔ Segmentation** — Consolidate vs fragment; bundle vs unbundle
6. **Universality ↔ Specialization** — Multi-function vs focused; platform vs point solution
7. **Nested ↔ Flat** — Hierarchy vs network; embedded vs standalone
8. **Anti-Weight ↔ Added Weight** — Lightweight vs heavy investment; lean vs robust
9. **Preliminary Anti-Action ↔ Reactive Action** — Proactive vs reactive; plan vs respond
10. **Preliminary Action ↔ Just-in-Time** — Pre-invest vs on-demand; reserve vs agile
11. **Beforehand Cushioning ↔ Lean/Bare** — Safety margin vs efficiency; buffer vs tight
12. **Feedback ↔ Feedforward** — Measure past vs predict future; reactive vs anticipatory

**Business Ideality** = Σ Customer Value / (Σ Costs + Σ Harms). All business TRIZ tools aim to increase this ratio.

## Part 7: 5 Thinking Components for Business (Ch19)

For each business problem, identify:
1. **The Ideal Final Result (B-IFR):** The business delivers maximum customer value with zero cost and zero harm
2. **The Contradiction:** The business parameter trade-off preventing the B-IFR
3. **Available Resources:** Assets, capabilities, data, relationships, brand, culture — everything already present that can be leveraged
4. **The Zone of Conflict:** Where in the value chain the contradiction physically/manifestly occurs
5. **The Trends:** Which Trend of Evolution applies to this business system

## Part 8: 10-Step Innovation Process (Ch20)

**PHASE I — STRATEGIC (Where & Why):**
1. **Situation Analysis:** Market, technology, competitive, patent landscape → current state map
2. **Strategic Diagnosis:** Function Analysis of business model, bottlenecks → highest-leverage innovation points
3. **Innovation Goal Setting:** IFR for business, measurable targets → "What exactly and by when?"
4. **Strategy Development:** Contradiction Matrix (business), Trends, S-Curve → innovation roadmap
5. **Strategic Controlling:** Balanced Scorecard, KPI dashboards → measurement system

**PHASE II — EXECUTION (What & How):**
6. **Idea Generation:** 40 Principles, Separation, Effects, Trends, AFI-inverted → 50-100+ raw ideas
7. **Idea Evaluation:** PUGH Matrix, Ideality assessment, feasibility → ranked top 5-10
8. **Concept Development:** ARIZ, Su-Field, Trimming → detailed designs, specifications
9. **Implementation:** Prototyping, testing, IP protection → validated, production-ready solutions
10. **Innovation Controlling:** Post-launch KPI tracking → feeds back to Step 1 (continuous loop)

**Mental model:** Innovation is a process, not an event. Strategy before execution, always. The output is decisions, not ideas. Step 10 feeds back to Step 1 — it's cyclical, not linear.

## Part 9: Customer Benefit Analysis — I-S Matrix (Ch21)

**Process:** (1) Identify benefit dimensions in customer language → (2) Measure Importance (I, 1-10) from CUSTOMERS → (3) Measure Satisfaction (S, 1-10) vs best available alternative → (4) Gap = I - S → (5) Translate largest gaps to TRIZ problem statements.

**The Matrix:**

| | High Importance | Low Importance |
|---|---|---|
| **Low Satisfaction** | **SWEET SPOT** — Priority #1, invest TRIZ effort here | **IRRELEVANCE** — Ignore |
| **High Satisfaction** | **PARITY** — Maintain, don't over-invest | **OVERSHOOT** — Trim investment |

**When:** Early in any innovation project; prioritizing TRIZ effort. **When NOT:** Pure technology-push with no existing customers.

## Part 10: Implementation Pipeline (Ch22)

5-stage structured bridge from concept to production:

1. **Concept Validation:** Minimum proof-of-principle. Answer: "Does the core mechanism work?" Budget: <5% of project.
2. **Detailed Design:** Resolve ALL secondary problems; engineering calculations; apply AFI; full specs.
3. **Prototyping & Testing:** Escalate: breadboard → engineering → production. Test under real conditions. AFI on test results.
4. **Production Ramp-Up:** Tooling, documentation, training, pilot run → full production.
5. **Post-Launch Monitoring:** Field performance, warranty, customer feedback → feeds next innovation cycle.

**Prototype Escalation Ladder:** Thought experiment → Simulation/CAE → Breadboard/3D-print → Engineering prototype → Production prototype.

**Key anti-patterns:** Jumping concept→production prototype; ignoring secondary problems; treating resistance as a people problem rather than design feedback.

## Part 11: 7 CAI Tool Categories (Ch23)

| Category | What It Does | When to Use |
|---|---|---|
| Contradiction Solvers | Auto-match parameters to Principles | Fast principle lookup from natural language |
| Effects Databases | Search physical/chemical/geometric effects by function | "How to achieve function X physically?" |
| Patent Analyzers | Mine patents for trends, white spaces, design-arounds | Patent landscape, competitor circumvention |
| Semantic Search | NLP match problem description → TRIZ knowledge | User knows problem but not which TRIZ tool |
| Technology Forecasting | S-Curve + Trend mapping | Predicting next-gen technology timing |
| Function Modelers | Build/analyze Function Models; auto-find contradictions | Complex system mapping |
| Value Engineering | Function Analysis + cost data → trimming targets | Cost reduction, value optimization |

**Function-Oriented Search (FOS)** is CAI's killer app: search across ALL industries for how a FUNCTION was achieved, not a component name. "How to move micro-litres of liquid?" finds solutions from medical devices, inkjet printers, and fuel injection simultaneously.
